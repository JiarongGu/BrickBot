# Doc Update Technical

**Format**: `/doc-update-technical <Document> <UpdateType> <Details>`
**Documents**: `ADVANCED_PATTERNS` | `DESIGN_DECISIONS`
**UpdateTypes**: `new-pattern` | `new-decision` | `update-pattern` | `deprecate-pattern`

## Action

1. Read the target doc (`docs/core/ADVANCED_PATTERNS.md` or `docs/core/DESIGN_DECISIONS.md`)
2. Based on UpdateType:
   - `new-pattern` → add section with: code example, **Why** rationale, **Trade-offs** (pros/cons)
   - `new-decision` → add entry with: Decision, Context, Alternatives, Trade-offs, Status: Active
   - `update-pattern` → update existing section, add version note
   - `deprecate-pattern` → add DEPRECATED marker, explain replacement
3. Update "Last Updated" date
4. Write the updated file

## Key Rule

Only document non-automatable patterns here. If a pattern can be generated by a skill, it belongs in the skill — not in these docs.
